    ```typescript
    type AddLinesInput = {
      companyId: string;
      countId: string;
      lines: Array<{
        itemId: string;
        cylinderState: CylinderState;
        countedQuantity: number;
        remarks?: string;
      }>;
    };
    ```
    - Fetch the PhysicalCount. Validate status is DRAFT (can only add lines to draft counts).
    - For each line:
      - Query the current ledger quantity at this location:
        ```
        const latest = await tx.stockLedgerEntry.findFirst({
          where: { companyId, itemId, cylinderState, locationId: count.locationId },
          orderBy: [{ transactionDate: "desc" }, { createdAt: "desc" }],
          select: { balanceAfter: true },
        });
        const ledgerQuantity = latest?.balanceAfter ?? 0;
        ```
      - Calculate variance = countedQuantity - ledgerQuantity.
      - Create PhysicalCountLine with ledgerQuantity, countedQuantity, variance.
    - Update count status to IN_PROGRESS.
    - Write audit log.
    - Return the created lines.

    **Function `approvePhysicalCount(tx, input)`:**
    ```typescript
    type ApproveCountInput = {
      companyId: string;
      financialYearId: string;
      userId: string;
      countId: string;
    };
    ```
    - Fetch the count with lines. Validate status is IN_PROGRESS (not already approved or cancelled).
    - For each line where variance !== 0:
      - If variance > 0 (counted more than ledger — stock surplus):
        Create IN entry:
        ```
        await createStockLedgerEntry(tx, {
          companyId, financialYearId,
          itemId: line.itemId,
          cylinderState: line.cylinderState,
          direction: StockDirection.IN,
          sourceType: StockSourceType.PHYSICAL_COUNT_ADJUSTMENT,
          sourceId: count.documentNo,
          transactionDate: count.countDate,
          quantity: Math.abs(line.variance),
          createdById: input.userId,
          locationId: count.locationId,
          remarks: `Physical count adjustment: counted ${line.countedQuantity}, ledger ${line.ledgerQuantity}`,
        });
        ```
      - If variance < 0 (counted less than ledger — stock shortage):
        Create OUT entry:
        ```
        await createStockLedgerEntry(tx, {
          companyId, financialYearId,
          itemId: line.itemId,
          cylinderState: line.cylinderState,
          direction: StockDirection.OUT,
          sourceType: StockSourceType.PHYSICAL_COUNT_ADJUSTMENT,
          sourceId: count.documentNo,
          transactionDate: count.countDate,
          quantity: Math.abs(line.variance),
          createdById: input.userId,
          locationId: count.locationId,
          remarks: `Physical count adjustment: counted ${line.countedQuantity}, ledger ${line.ledgerQuantity}`,
        });
        ```
    - Update count status to APPROVED, set approvedById, approvedAt.
    - Write audit log with summary of adjustments created.
    - Return the updated count.

    **1. Create `src/server/services/reports/stock-by-location.ts`:**

    Export `getStockByLocation`:
    ```typescript
    type StockByLocationInput = {
      companyId: string;
      financialYearId: string;
      locationId?: string; // optional filter for single location
    };

    type StockByLocationRow = {
      locationId: string;
      locationCode: string;
      locationName: string;
      itemId: string;
      itemCode: string;
      itemName: string;
      filledQuantity: number;
      emptyQuantity: number;
    };

    export async function getStockByLocation(tx: Prisma.TransactionClient, input: StockByLocationInput): Promise<StockByLocationRow[]>
    ```

    Implementation:
    - Query the latest balanceAfter for each (locationId, itemId, cylinderState) combination from StockLedgerEntry.
    - Use raw Prisma approach: for each cylinderState (FILLED, EMPTY), use `findFirst` ordered by transactionDate desc + createdAt desc, OR use a `groupBy` approach.
    - Most efficient: run a single query per state using `stockLedgerEntry.groupBy` with `by: ["locationId", "itemId"]` and `_sum: { quantity: true }` with `direction: IN → +1, OUT → -1`. Or use `findFirst` with `groupBy` on the subquery.
    
    Actually the simplest approach matching existing patterns:
    - Fetch the latest balance for each location+item+state combination using the ledger's `balanceAfter` column (it's designed to be the running balance).
    - Use `findFirst` grouped:
    ```
    // For each unique (locationId, itemId) pair, get the latest FILLED and EMPTY balance
    // Use raw SQL or a series of Prisma queries
    ```
    
    For v1 simplicity, use a pragmatic approach: 
    1. Get all distinct (locationId, itemId) combos from StockLedgerEntry for this company+financialYear
    2. For each combo, get the latest FILLED balance and latest EMPTY balance
    3. Join with StockLocation and Item for names
    
    BUT to keep it performant, use `groupBy` with a signed sum (same pattern as stock-availability.ts):
    ```
    const rows = await tx.stockLedgerEntry.groupBy({
      by: ["locationId", "itemId", "cylinderState", "direction"],
      where: {
        companyId: input.companyId,
        financialYearId: input.financialYearId,
        locationId: input.locationId ?? undefined,
      },
      _sum: { quantity: true },
    });
    ```
    Then compute net per (locationId, itemId, cylinderState) = sum(IN) - sum(OUT).
    
    This is simpler and uses the same pattern as `getFilledStockByItem`. The only tradeoff is it recomputes from all entries rather than using cached balanceAfter. At current data volumes this is fine.

    - After computing the map, fetch location names from StockLocation and item names from Item for the response.
    - Return as array of StockByLocationRow.
